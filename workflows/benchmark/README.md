# Product trials

`npm run check` covers deterministic contracts. It is not evidence that this machine's systemd,
Bubblewrap, Btrfs, provider, or network paths work together.

Before calling a build production-ready, run fresh-project trials for:

- structured research completion;
- candidate editing with mediated research;
- a receiptless yield followed by successful session reopen;
- a three-strike pause with the session and candidate retained;
- `builtin:goal` with a deliberate handoff;
- `builtin:execute-plan` with multiple points;
- apply approval and rejection;
- extension reload during an asynchronous run;
- coordinator and agent `SIGKILL` recovery;
- provider and network interruption;
- same-run replay and explicit cross-revision prefix replay.

For every trial retain the run ID, workflow source hash, route snapshot, operation/event pages,
agent finish receipt, artifact digests, workspace checkpoint, systemd unit result, cgroup metrics,
and final apply decision. Record snapshot, checkpoint, command-start, projection, and replay-hit
latency directly from the run evidence. Never infer success from assistant prose.
